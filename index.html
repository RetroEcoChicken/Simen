This video shows me using a filesystem that i build from scratch on a VM running my OperatingSystem.
It can currently only handle a single sector of the harddrive so im limited to only 500 char's
however its still a big step from not beeing able to store anything on the harddrive and i still learn how to
manipulate files and parse them.

My operating system might be very basic but it does come with a norwegian keyboard driver as default.
It also has some barebones networking protocols but im still not able to ping websites.

The @B command stores everything behind it into a buffer that the kernel can then use.

The brackets {} tells the kernel that i want to execute a built in function and then the kernel searches for
a function of that name and expects the user to have set up the preperation steps if there is any.

{write} and {read} are functions built in to do direct requests from the ata driver.

This is how i setup another buffer for dynamic file creation[filename][data]
It also formats the data so its ready to be written

The {addfile} function feeds the formatted data into the harddrive. It first gets the already existing data,
then adds the new data then writes everything to disk. This is because when you write you will overwrite
existing data.
